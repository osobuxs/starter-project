# DocsV2

Esta carpeta es la **fuente de verdad documental** de la versión final implementada del proyecto.

## Cómo leer esta documentación

En este repositorio conviven tres capas de contexto:

1. **Starter original**: la base de noticias que ya existía al inicio.
2. **Challenge original**: lo que se pedía resolver sobre ese starter.
3. **Implementación final**: lo que efectivamente se construyó y endureció en esta entrega.

La documentación legacy del root, `docs/` y parte de `backend/` se conserva como contexto histórico, pero puede no reflejar el estado final real del producto.

## Qué vas a encontrar acá

- `01-overview/` → visión general y alcance real
- `02-scope/` → comparación entre starter, challenge original e implementación final
- `03-frontend/` → arquitectura y navegación actuales
- `04-backend/` → contrato real de Firestore/Storage/rules/indexes
- `05-delivery/` → documentación pensada para evaluadores

## Qué archivo conviene leer primero

Si sos evaluador y querés entender rápido el proyecto, empezá por:

1. `02-scope/IMPLEMENTATION_MATRIX.md`
2. `03-frontend/ARCHITECTURE.md`
3. `04-backend/FIREBASE_CONTRACT.md`

## Nota importante

Esta carpeta no intenta esconder el starter original ni “reescribir la historia”. Al contrario: deja explícito qué ya existía, qué se pedía, qué se implementó y qué se agregó como overdelivery.
