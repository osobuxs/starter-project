# Database Schema

## Overview

The app uses **Cloud Firestore** as its primary database and **Firebase Cloud Storage** for media files. Authentication is handled by **Firebase Authentication** (email/password).

---

## Collections

### `users/{uid}`

Stores the editable profile of each authenticated user. The `uid` matches the Firebase Auth UID.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | String | Yes | Display name |
| `email` | String | Yes | User email (mirrors Firebase Auth) |
| `age` | Number | No | User age |
| `photoUrl` | String | No | Firebase Storage download URL |
| `createdAt` | Timestamp | Yes | Account creation time |
| `updatedAt` | Timestamp | Yes | Last profile update time |

---

### `articles/{articleId}`

Stores journalist-published articles. The `articleId` is auto-generated by Firestore.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `title` | String | Yes | Article title |
| `content` | String | Yes | Full article content |
| `category` | String | No | Article category |
| `thumbnailUrl` | String | No | Firebase Storage URL (`media/articles/{articleId}/thumbnail.jpg`) |
| `authorId` | String | Yes | Firebase Auth UID of the author |
| `authorName` | String | Yes | Author display name at publish time (denormalized) |
| `authorPhotoUrl` | String | No | Author avatar URL at publish time (denormalized) |
| `createdAt` | Timestamp | Yes | Publication timestamp — used for ordering (descending) |
| `updatedAt` | Timestamp | Yes | Last update timestamp |
| `isPublished` | Boolean | Yes | Whether article is visible in the dashboard |
| `isActive` | Boolean | Yes | Soft delete flag — `false` means logically deleted |
| `savedCount` | Number | Yes | Total users who favorited this article (default: `0`) |
| `tags` | Array\<String\> | No | Optional content tags |

---

### `users/{uid}/favorites/{articleId}`

Subcollection that tracks which articles a user has saved as favorites. The `articleId` matches the document ID in `articles/`.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `articleId` | String | Yes | Reference to `articles/{articleId}` |
| `savedAt` | Timestamp | Yes | When the user saved this article |

---

## Firebase Cloud Storage Structure

```
media/
  users/
    {userId}/
      profile.jpg          ← User profile photo
  articles/
    {articleId}/
      thumbnail.jpg        ← Article thumbnail image
```

---

## Firestore Indexes

The following composite indexes are required for dashboard queries:

| Collection | Fields | Order | Purpose |
|------------|--------|-------|---------|
| `articles` | `isActive` ASC, `isPublished` ASC, `createdAt` DESC | — | Dashboard default load |
| `articles` | `isActive` ASC, `isPublished` ASC, `createdAt` ASC | — | Date range filter (start boundary) |
| `articles` | `authorId` ASC, `createdAt` DESC | — | "Mis notas" query |

---

## Security Rules Summary

| Resource | Read | Write | Condition |
|----------|------|-------|-----------|
| `articles` (active) | Anyone | — | `isActive == true AND isPublished == true` |
| `articles` | — | Auth users only | `request.auth != null` |
| `articles/{id}` update/delete | — | Author only | `request.auth.uid == resource.data.authorId` |
| `users/{uid}` | Owner only | Owner only | `request.auth.uid == uid` |
| `users/{uid}/favorites` | Owner only | Owner only | `request.auth.uid == uid` |
| `media/users/{userId}/` | Public | Owner only | `request.auth.uid == userId` |
| `media/articles/{articleId}/` | Public | Auth users | `request.auth != null` |
